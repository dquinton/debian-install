The following method is ideal for building a minimal Debian install (without bloat) as it can install just the basic system without a desktop environment or any software packages, following which, you boot into a command prompt where you can install whatever else is needed for you to use Debian, through an internet connection. The Netinstall CD does give you the choice to install a full Desktop Environment as well, if you choose to, but then you can't specify exactly what you want installed at that level.

Another method is to get a Debian Live CD with a full DE of your choice, if you prefer, but then you lose the fun of keeping the install cut down to exactly what packages you want to be installed on your box.

There are different releases of Debian - Experimental (just don't, because there will be many breakages of your system), Unstable (don't either, there will still be breaks), Testing (where packages from new release are sent down to before becoming stable, will still be bugs), and Stable (where packages are well-tested and bugs mostly already repaired).

Jessie became the new Debian stable in 2015.. Debian 8. For better package updates, you can follow Debian testing, by adding "stretch" or "testing" to your package sources list instead of "jessie" (I'll show how below). If you add "testing", your Debian will stay with testing even when Stretch becomes the new Stable, Debian 9.

However, when Testing is still new in life there may be some things that could break your system (meaning you might have crashes or package install conflicts to sort out)... so running stable at the moment should be very sufficient, and highly recommended for production systems or servers. You can still install some packages from testing as well, by using the "-t testing" option with apt-get, while your default release is set to stretch or stable.

Upgrading to Testing, or Stretch, later on is also possible - see my page upgrade debian.

Prepare to install

The netinstall CD image that you have downloaded and burned to USB stick or CD will install only the very basic Debian Linux system and all the packages necessary to build a desktop environment and run programs will be retrieved afterwards, at your own leisure.

You need to have an internet connection available, either by ethernet cable or with a wireless network (so long as you have the ESSID, password and the wireless firmware files), as the installer will download most of the system and APT package lists.

The installer may ask for missing firmware, such as for wireless. You can get a zipped archive of firmware from Debian, and simply copy it to the USB stick with the Live ISO and unpack it there. The installer will search for what it wants. https://www.debian.org/releases/stable/amd64/ch06s04.html.en

Otherwise, the wireless and other firmware can be loaded after Debian is installed (see below).



Get a partition ready!

If you have a brand new PC to install on, or one with nothing worth keeping on the disk, you can format the harddisks with the installer - just go to no. 2 below. On a PC which already has an OS installed, use Gparted to make free space for your new partitions ->
Get the Gparted Live CD or install it if you have a Linux OS already on the PC. Then...
  1. shrink a large partition which has some unused space, to make enough for your combined system and home area, perhaps 30 or 40Gb (see below about resizing Windows).

    2. Note: it's not advisable to move a partition along on the hdd, as Gparted will actually copy the whole file system, sector by sector, taking a very long time. Shrinking from the right should be fast, but if you need to move the partition along to the right, it would be better to copy the data off it, delete it and then create new partitions.

  2. make an Extended partition =total size of the partitions needed for Linux, or else resize existing Extended partition to add Logical partitions to it.
  3. make a Logical partition for Linux root, ~15Gb (generous) formatted with ext4, another one for the home partition (if you choose to make it separate), anything from 2Gb to 20Gb... formatted ext3 or ext4 also.
  4. then a swap area (but no need to make if there's an existing swap of another Linux install) which just needs 1Gb to work fine in my opinion and formatted as Linux swap.


Windows dual-boot

If you are planning to dual boot Debian with Windows (i.e. have both installed and choose from Grub menu which one to boot) it will be easy as chips if your Win OS partition was custom sized with separate data partitions - then you just need to resize any data partition to make space for Debian. However, if your PC came with factory-installed Windoze, then the OS partition may indeed take up half or even the whole of the hard-disk. In that case, you'll need to run some operations in W so that you can shrink the OS partition down to a more moderate 50 to 80Gb.

I ran the following tasks in Win7 to do this:
  • disk clean-up
  • disable "page file" and "kernel memory dump" [system>advanced tab]
  • turn off system restore [system>system protection]
  • turn off hybernation [power>change when computer sleeps>sleep>hibernate>0]
  • turn off search feature [prog and features>turn windows features on or off]
  • turn off indexing for drives [rightclick drive>properties>uncheck index>click Apply>click Ignore all]
  • defrag [admin tools]
  • attempt to shrink partition in disk management.

Finally I shrunk it right down to the size I wanted it with Gparted (but you may want to back up the partition before doing this).

If you're going to to wipe the W partition (yeah!) but there's data on it, back-up what you need to external drive or data partition if present, or else shrink down the W partition and make a temporary partition to move stuff onto.

Take care if you actually shift a Windows (Win 7) partition along the HDD, say to make more space for a Linux partition, as this will LIKELY render the Windows OS unbootable because the W boot sector will be changed. You would then need to fix that with a Windows Disk by running the repair boot option.


Debian installer might not see the Win OS and so you will need to do some work to add the Windows OS to the boot menu later. Some tips on my GRUB page - although I only cover Legacy GRUB. Windows 8 boots with UEFI boot menu which makes it complex to have Debian on the same menu (see here for example). With Debian on the Legacy menu you'll have to switch between Legacy and UEFI menus in BIOS settings.

The install

1. I chose Graphical expert install but plain Graphical install does the job and it might be faster.

2. configure network hardware: insert usb stick at firmware prompt for network hardware (or if it's on your Live CD stick just wait).

3. then set up the network by either Ethernet (eth0) or wireless (wlan0), if you have the ESSID ready (your wireless network name)

4. Partitioning:
  • choose Manual, go down the list to select the new root partition,
  • press Enter on "do not use" and change to Ext4 journaling file system (or what you previously formatted it to)
  • set mount point to /
  • set bootable flag to on
  • do not format (unless a previous install on it was unsuccessful)
  • choose the home partition and tap again on "do not use" change it to ext3; mount as /home
  • choose the swap if it's the first Linux install on the machine and set it to swap.
  • Finish partitioning

5. install software.. uncheck all (tap Spacebar) including graphical desktop (which installs the bloated Gnome, or Xfce with a lot of extras), select laptop if installing to a laptop.

6. Install GRUB (if for clean install) or otherwise don't - add the new install to GRUB later. To do that I used "Grub Customizer" loaded on another Debian OS on the same box but there are other ways to do it, "Boot repair disk" being a good way too. (see my GRUB page)

-the netinstaller didn't detect my other OS so couldn't set up grub.

Then finish the install to set-up users and passwords... restart and boot into a system with, as yet, no desktop (it will be just a command prompt).

Install Thunar file manager

Thunar is the best light-weight gui file browser in my opinion, beating the default LXDE file manager, because of its ability to carry out possibly any action whatsoever on files through its right-click menu. There are more powerful file browsers about, such as Dolphin or Krusader (both KDE), but you should look at this one first, and see my custom actions on my thunar page. For example, Krusader can do some picture resizing actions - but you need the KIO plugins installed, whereas with thunar, all you need is imagemagic (a small program) and a few custom actions and you can resize pic's with two clicks - or resize a whole directory of pic's, even recursively.

There are alternative file browsers too, such as xfe (with no bookmarking except your own real symlinks), gentoo (two pane with task buttons), emelfm2 (lots of task buttons), and rox-filer (very basic graphical).

But for those bored of GUI mouse-clicking browsers there's Ranger (which takes up my 2nd desktop in full screen) - see my ranger page.
